# Redis 短记忆 与 mem0 长记忆

## 1. 核心区别

```txt
Redis = 短期记忆，保存当前任务运行状态
mem0  = 长期记忆，保存跨会话可复用经验
```

一句话：

```txt
Redis 管过程
mem0 管经验
```

---

## 2. Redis 短记忆

Redis 适合保存当前任务里的临时状态。

例如：

```txt
当前轮次 round
最近 intent
已读文件 readFiles
已修改文件 changedFiles
工具调用记录
noProgressRounds
failedCount
当前 mode
当前 hypothesis
临时 context summary
任务锁 / 并发锁
```

特点：

```txt
生命周期短
读写频繁
任务结束后可以删除
主要服务当前 loop
```

可以理解成：

```txt
Redis = Agent 当前任务的草稿纸
```

示例：

```json
{
  "taskId": "task_001",
  "userInput": "登录有 bug",
  "round": 3,
  "mode": "react",
  "readFiles": [
    "src/pages/login.vue",
    "src/api/auth.ts"
  ],
  "changedFiles": [],
  "noProgressRounds": 1,
  "failedCount": 0,
  "recentIntents": ["read", "read", "inspect"]
}
```

---

## 3. mem0 长记忆

mem0 适合保存长期有价值的信息。

例如：

```txt
用户偏好
项目规则
稳定的项目结构
历史修复经验
跨会话可复用知识
用户明确要求记住的内容
```

特点：

```txt
生命周期长
跨任务可用
跨会话可用
通常通过语义检索召回
不应该每轮都写入
```

可以理解成：

```txt
mem0 = Agent 的长期经验库
```

示例：

```json
{
  "type": "project_rule",
  "content": "这个项目登录态保存在 src/store/user.ts，token 由 src/utils/request.ts 注入请求头。",
  "confidence": 0.86
}
```

---

## 4. 它们在 Agent 里的位置

```txt
User Input
  ↓
读取 Redis 短期状态
  ↓
检索 mem0 长期记忆
  ↓
RAG / 多路召回项目上下文
  ↓
Build Context
  ↓
Planner
  ↓
Tool Runner
  ↓
Observer
  ↓
更新 Redis
  ↓
必要时写入 mem0
```

核心规则：

```txt
每一轮都可以读写 Redis
只有重要结论才写入 mem0
```

---

## 5. 用“登录有 bug”举例

用户输入：

```txt
登录有 bug
```

Redis 记录当前任务过程：

```txt
读过 login.vue
读过 auth.ts
当前第 2 轮
最近 intent 是 read
还没有修改文件
```

mem0 召回长期经验：

```txt
这个项目 token 存在 user store
请求拦截器在 request.ts
用户偏好先读代码再修改
```

任务结束后，如果发现稳定结论：

```txt
登录 bug 常见原因是 request.ts 没有正确注入 Authorization
```

可以写入 mem0。

但这些不要写入 mem0：

```txt
某一轮 read_file 的完整内容
一次性的报错日志
临时 failedCount
临时 noProgressRounds
没有验证的猜测
```

---

## 6. 判断放哪里

| 内容        | 放 Redis | 放 mem0 |
| --------- | ------- | ------ |
| 当前轮次      | ✅       | ❌      |
| 最近 intent | ✅       | ❌      |
| 已读文件      | ✅       | ❌      |
| 连续失败次数    | ✅       | ❌      |
| 用户长期偏好    | ❌       | ✅      |
| 项目稳定规则    | ❌       | ✅      |
| 已验证的修复经验  | ❌       | ✅      |
| 临时报错日志    | ✅       | ❌      |
| 未验证猜测     | ✅       | ❌      |

---

## 7. 写入 mem0 的规则

不要每轮都写 mem0。

推荐规则：

```txt
用户明确要求记住 -> 写入 mem0
已验证 + 可复用 -> 写入 mem0
只是当前任务过程 -> 放 Redis
没有验证的猜测 -> 不写 mem0
```

伪代码：

```ts
function shouldWriteLongMemory(item: {
  verified: boolean;
  reusable: boolean;
  userExplicit: boolean;
}) {
  if (item.userExplicit) return true;

  return item.verified && item.reusable;
}
```

```txt
memory/

├── long-memory.ts    // mem0
├── memory-router.ts  // 判断读写哪个 memory
└── memory-policy.ts  // 判断什么值得长期保存
```

---

## 9. 总结

```txt
Redis 短记忆：
保存 Agent 当前正在做什么。
```

```txt
mem0 长记忆：
保存 Agent 以后还用得上的经验和偏好。
```

最短记法：

```txt
Redis = 当前任务状态
mem0  = 跨会话长期记忆
```
